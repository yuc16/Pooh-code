# Paper Research

## 目标

围绕学术研究主题，输出可用于写论文的研究材料，而不是泛泛的网页摘要。

优先完成这些事情：

1. 找到高相关论文
2. 区分经典论文与近年论文
3. 给出结构化来源信息
4. 输出可复核的论文说明与证据摘要

## 默认流程

### 1. 先澄清检索目标

先确认以下要素：

- 研究主题是什么
- 用户要的是综述、相关工作、背景介绍，还是找某篇具体论文
- 是否有限定年份、领域、开放获取、代表作者或期刊

如果用户没有给时间范围：

- 默认同时覆盖“经典高被引”与“近 3-5 年最新研究”

### 2. 优先调用 `paper_search`

涉及论文、文献、综述、参考文献时，默认先调用：

```text
paper_search(query=..., ...)
```

不要一开始就先用 `web_search`。

优先策略：

- 第一轮：用用户原始主题词直接搜
- 第二轮：如果结果太散，补充更具体的技术关键词
- 第三轮：必要时拆成多个子主题分别检索

常用参数建议：

- 要近年研究：`from_year`
- 要高被引经典文献：`sort="cited_by_count"`
- 要最近进展：`sort="publication_year"`
- 用户明确只要可读全文：`open_access_only=true`

### 3. 再决定是否补网页阅读

当 `paper_search` 已返回较完整元数据时：

- 优先使用返回的 `abstract`
- 需要更深入时，再访问 `landing_page_url`
- 如果有 `pdf_url` 或开放获取页面，再用 `web_fetch` 补读正文页

只有当 `paper_search` 找不到足够结果，或用户明确要求网页资料/新闻资料时，才退回：

- `web_search`
- `web_search_and_read`

### 4. 做文献分层

输出前，至少把结果分成以下一类或多类：

- 经典基础文献
- 近年代表性工作
- 与用户问题最直接相关的论文
- 可进一步精读的高价值论文

不要只丢一个混杂列表。

### 5. 产出引用友好的结果

回答里尽量保留这些字段：

- 标题
- 作者
- 年份
- 期刊 / 会议 / venue
- DOI 或 OpenAlex / 落地页链接
- 被引数

如果用户要求“参考文献”或“引用格式”，优先输出规范化条目；如用户未指定格式，默认给“论文说明块 + DOI/链接”的结构，而不是把参考文献全部堆到最后。

## 输出要求

### 总体格式

涉及论文研究、相关工作、文献综述、参考文献时，默认使用下面这套输出骨架：

1. 先给结论或综述正文
2. 每篇论文在被解释时，直接附 DOI 或可访问链接
3. 如果用了编号引用，例如 `[1]`、`[2]`，编号对应的论文说明块应紧跟在相关小节后面出现
4. 默认不再强制在文末单独追加 `参考文献` 小节

不要只给一堆 URL，也不要只给论文标题不附来源信息。

### 引用规则

- 正文中的事实判断、研究结论、方法归类、代表性工作举例，可以带编号引用
- 一个段落如果综合了多篇论文，句末可并列引用，例如 `[1][3][5]`
- 但编号不应只在文末解释，相关论文说明块里必须直接出现 DOI 或落地页链接
- 如果某条信息只来自摘要而不是全文，必要时可在句中说明“基于摘要判断”
- 如果论文元数据不完整，必须在对应论文条目里标 `待核验`
- 不要编造页码、卷号、期号、出版社等 OpenAlex 未明确给出的字段

### 论文条目写法

优先直接复用 `paper_search` 返回结果中的：

- `inline_citation`
- `reference_text`

但默认输出时，不要把 `reference_text` 原样挪到最后集中列出；应优先拆进每篇论文介绍块里。

默认每篇论文至少写：

- 标题
- 作者
- 年份
- 期刊 / 会议 / venue
- 价值 / 为什么和用户主题相关
- DOI / 链接

如果用户没有明确指定格式，不必强行输出严格 APA/MLA/BibTeX；默认输出“论文说明块 + 就地链接”即可，但必须可复核。

### 如果用户要“帮我研究一个方向”

优先输出：

1. 研究方向概览
2. 代表性论文分组
3. 每篇论文一句话价值说明
4. 建议优先精读的 3-5 篇
5. 每篇论文条目内直接附 DOI / 链接

### 如果用户要“帮我写 related work / 文献综述”

优先输出：

1. 按研究路线分组
2. 每条路线的代表论文
3. 路线差异与演进关系
4. 可直接写进论文的综述段落草稿
5. 段落中可以用 `[1]`、`[2]` 这种编号引用
6. 但编号对应的论文说明块要紧跟在该路线小节后，并直接附 DOI / 链接

### 如果用户要“给我参考文献”

至少输出：

- 规范化文献列表
- 每条的 DOI 或链接

如果用户明确说要“直接能贴进论文”，优先输出：

- 一段带编号引用的正文
- 再附对应的论文说明块
- 每个说明块里直接放 DOI / 链接

## 重要边界

- 不要把普通网页、博客、新闻伪装成学术论文
- 不要捏造 DOI、作者、年份、期刊名
- 若某论文信息不完整，要明确写“待核验”
- 若只有摘要没有全文，不要假装读过全文
- 对论文贡献的描述应尽量基于摘要、元数据或已读到的正文证据

## 简化决策规则

- 学术问题 → `paper_search` 优先
- 已有具体论文页面 → `web_fetch`
- 需要补更多背景网页资料 → `web_search` / `web_search_and_read`
