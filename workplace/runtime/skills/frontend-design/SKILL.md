# Frontend Design

## 目标

产出具有明确审美方向、可直接运行的前端代码，而不是泛泛设计建议。

适用范围：

- 页面
- 组件
- 小型应用界面
- 落地页
- 仪表盘
- 品牌展示型前端

所有面向用户的文案默认使用**简体中文**，除非项目协议或现有代码明确要求英文。

## 使用原则

在开始编码前，先快速明确这四件事：

1. **用途**：这个界面解决什么问题，服务谁
2. **审美方向**：必须选择清晰风格，不要中庸混搭
3. **技术约束**：框架、性能、响应式、可访问性要求
4. **记忆点**：这个设计最让人记住的一点是什么

可选风格方向示例：

- 极简冷峻
- 浓烈杂志感
- 复古未来
- 工业功能主义
- 柔和有机
- 奢华精致
- 粗野主义
- 几何装饰风
- 玩具感 / 趣味感

重点不是“堆效果”，而是**选定方向并执行到底**。

## 输出要求

产出的实现必须同时满足：

- 代码真实可运行
- 视觉上有明确个性
- 细节统一，不东拼西凑
- 符合当前项目技术栈
- 不落入常见 AI 模板审美

## 前端审美执行准则

### 1. 字体

- 避免默认套用 Arial、Roboto、Inter、系统默认无衬线作为唯一方案
- 优先选择更有识别度的标题字体，并搭配克制的正文字体
- 字体选择要服务整体气质，而不是为了“特别”而特别

### 2. 配色

- 使用 CSS 变量或设计 token 保持一致性
- 主色要明确，强调色要克制但有冲击力
- 避免常见的“白底 + 紫蓝渐变 + 通用卡片”套路

### 3. 动效

- 动效用于建立气质和层次，不是为了炫技
- 优先做好少数关键时刻：首屏进入、悬停反馈、滚动揭示
- HTML/CSS 场景优先 CSS 实现；React 场景在项目已有能力时可用 Motion 类库

### 4. 版式与空间

- 可以使用非对称、错位、叠层、超出版心等手法建立记忆点
- 同样也可以选择克制的留白体系，但必须足够精确
- 不要默认套用千篇一律的卡片栅格

### 5. 背景与细节

- 背景应参与氛围塑造，而不只是填充底色
- 可结合项目场景使用：纹理、噪点、网格、渐变网面、描边、阴影、透明层、装饰图形
- 所有效果都应服务整体概念，避免无意义堆叠

## 明确禁止

不要输出这些常见问题：

- 过度模板化的 AI 风格页面
- 没有主题的随意混搭
- 只换颜色不换结构
- 只强调“高级感”但没有具体设计动作
- 布局、字体、配色都落回常规默认值

## 执行方式

当用户要求你实现前端界面时：

1. 先用一句话确认你选择的审美方向
2. 再给出真实代码，而不是停留在概念描述
3. 必要时补充关键交互说明和组件拆分思路
4. 保持风格一致，避免做成“功能有了但视觉普通”的结果

## 最终标准

用户拿到的结果应当：

- 能直接继续开发或预览
- 有明确风格，不像通用脚手架页面
- 在字体、色彩、布局、动效、细节上彼此呼应
- 让人能一眼记住这个界面，而不是看完就忘
