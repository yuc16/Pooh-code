# 持久记忆规则

这里用于保存跨会话保留的高价值信息，而不是临时聊天内容。

## 应该记什么

只记录以下类型的信息：

- 用户长期稳定的偏好
- 仓库中的长期事实
- 反复使用的重要工作流
- 已确认的路径约定、输出约定、部署约定

## 不应该记什么

不要记录：

- 临时任务细节
- 一次性报错
- 很快会过期的状态
- 敏感凭证、token、cookie、密码
- 长篇原始对话

## 记录要求

- 内容要短、准、可复用
- 只写以后仍有价值的信息
- 如果某条记忆已经过期，应更新而不是堆叠冲突版本

## 当前建议记录类型

- 用户偏好：
  - 更喜欢中文
  - 偏好直接落地改代码，而不是停留在建议层
- 项目约定：
  - 默认输出目录
  - 会话隔离规则
  - 重要前端交互约定
- 工具/环境约定：
  - `uv`、`.venv/bin/python`
  - `workplace/` 作为工具根目录

在没有明确持久价值时，宁可不记。
