# 子代理使用约定

`pooh-code` 支持在主代理上下文外启动子代理，用于处理边界清晰、可以并行或降上下文压力的子任务。

## 何时使用子代理

- 需要对仓库做局部检索、代码定位、结构梳理，但不希望主上下文被大量搜索结果占满
- 需要把一个大任务拆成主线执行 + 辅助分析
- 需要快速回答“某个模块在哪里、某个函数做了什么、某个文件是否被引用”这类局部问题

## 当前可用类型

- `explorer`
  - 默认首选
  - 适合代码搜索、仓库分析、只读调查
  - 子代理以只读模式运行
- `general`
  - 适合更一般的受限子任务
  - 仍应保持任务边界清晰，避免把整个主任务外包出去

## 调用原则

- 只有当子任务明确、可独立完成时才启动
- 主代理负责最终整合，不要把最终答复完全交给子代理
- 优先让子代理返回结论、定位信息、候选修改点，不要返回冗长复述
- 不要为同一件小事连续启动多个重复子代理
- 子代理只服务当前主任务，不承担长期记忆角色

## 任务设计建议

- 描述要具体
  - 好例子：定位 Web 前端中消息渲染入口和复制按钮相关代码
  - 差例子：帮我看看整个项目
- 提示里要写清目标产物
  - 例如：返回文件路径、函数名、修改建议或结论摘要
- 如果主代理已经掌握足够上下文，就不要再开子代理

## 输出要求

子代理返回内容应尽量简短，优先包括：

- 关键结论
- 相关文件路径
- 必要时给出下一步建议

## 限制

- 子代理不会继续再开新的子代理
- 主代理必须保留对最终修改和最终回答的控制权
- 涉及文件编辑时，除非确有必要，否则优先由主代理执行最终改动

## 手动入口

用户可以通过命令手动触发子代理：

```text
/subagent <task>
```

该入口当前默认使用 `explorer` 类型。
